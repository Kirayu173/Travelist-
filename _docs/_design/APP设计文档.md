# 智能旅游APP设计说明书

## 一、项目概述

### 1.1 项目背景

随着旅游业的快速发展和移动互联网的普及，用户在旅行前后对**个性化行程规划、实时信息获取与智能交互体验**的需求不断增长。传统旅游应用多侧重于信息展示与导航，缺乏智能化的行程定制与即时辅助功能。

本项目旨在设计并开发一款融合智能规划、地图探索与AI助手功能的 **智能旅游APP** ，为用户提供全流程的智慧出行体验。系统通过整合出行规划、兴趣点发现与智能问答，构建一个高交互性、高个性化的旅游生态系统。

### 1.2 项目目标

* 为用户提供 **高效、灵活、个性化的旅行路线规划服务** ；
* 通过地图展示与兴趣点推荐，帮助用户 **发现周边精彩** ；
* 借助智能助手实现行程中 **即时信息支持与语义交互** ；
* 构建可持续扩展的系统结构，为后续功能拓展与智能升级奠定基础。

---

## 二、总体设计

### 2.1 系统功能结构

系统整体由三大核心模块构成：

1. **旅行路线规划模块**
2. **发现模块（地图与兴趣点推荐）**
3. **行程助手模块（智能体辅助）**

三者之间形成“ **行前规划 → 行中探索 → 行程辅助** ”的闭环体验。

```
用户入口
 ├── 旅行路线规划
 │    ├── 自定义行程
 │    ├── 智能生成行程
 │    ├── 行程卡片编辑与拖拽
 │    └── 地图展示与导航跳转
 │
 ├── 发现模块
 │    ├── 实时定位展示
 │    ├── 兴趣点推荐与筛选
 │    ├── 兴趣点详情查看
 │    └── 智能问答与导航跳转
 │
 └── 行程助手模块
      ├── 智能问答
      ├── 天气预报
      └── 实时行程辅助服务
```

## 三、功能模块设计

### 3.1 旅行路线规划模块

#### 3.1.1 功能描述

该模块是系统的核心，负责帮助用户**创建、管理与智能生成**旅行行程。

#### 3.1.2 主要功能

1. **自定义行程**
   * 用户可选择目的地与旅行天数；
   * 每一天生成一张“行程卡片”；
   * 每张行程卡片包含若干“子行程”（活动、地点、备注）；
   * 子行程之间可定义交通方式（步行、骑行、驾车、公共交通）；
   * 支持用户自由创建、删除子行程；
   * 支持拖拽调整顺序，系统自动更新交通信息。
2. **行程卡片排序**
   * 用户可通过拖拽调整行程卡片顺序（如调换第1天与第2天）；
   * 系统自动调整日期索引与展示顺序。
3. **智能规划模式**
   * **普通模式** ：基于规则与算法（距离、时间、兴趣权重）快速生成建议行程；
   * **深度思考模式** ：调用大语言模型（LLM）生成每一天的行程建议；
   * 生成后用户可编辑与调整，支持混合编辑与保存。
4. **智能助手浮窗**
   * 页面右下角悬浮气泡；
   * 用户可通过自然语言询问如“推荐第一天去哪玩？”、“帮我加个晚餐安排”等问题；
   * 支持上下文记忆与行程修改建议。
5. **地图展示（可选增强）**
   * 地图展示每张行程卡片的地理位置与路线；
   * 可切换显示不同日期行程；
   * 支持跳转至第三方导航App（如高德、百度）。

---

### 3.2 发现模块

#### 3.2.1 功能描述

该模块负责在用户授权定位后，展示当前位置与周边兴趣点，实现行中探索体验。

#### 3.2.2 主要功能

1. **实时定位**
   * 自动获取用户当前位置；
   * 地图实时显示用户位置标识。
2. **兴趣点展示**
   * 通过地图展示周边美食、景点、酒店等兴趣点；
   * 不同类型以不同图标或表情表示；
   * 用户可筛选兴趣点类型与范围。
3. **智能助手交互**
   * 右下角浮窗可询问“附近有好吃的餐厅吗？”、“帮我找景点”；
   * 智能体返回筛选结果或推荐信息。
4. **兴趣点详情**
   * 点击兴趣点弹出详细信息；
   * 显示图片、地址、简介、评分等；
   * 可收藏或添加至行程卡片。
5. **一键导航**
   * 支持跳转第三方导航App；
   * 保持行程与位置数据同步更新。

---

### 3.3 行程助手模块

#### 3.3.1 功能描述

行程助手是系统的智能中枢，负责在用户出行中提供多样化的信息与辅助服务。

#### 3.3.2 功能内容

1. **基础问答**
   * 提供通用问答与信息检索；
   * 如“明天的行程是什么？”、“附近有超市吗？”等。
2. **天气预报**
   * 获取用户行程地点的实时天气；
   * 提供温度、降水、空气质量等信息；
   * 可主动提醒天气异常或出行注意事项。
3. **可拓展功能**
   * 交通拥堵预测；
   * 酒店/门票预订接口；
   * 实时行程提醒与语音播报。

## 四、系统架构设计（逻辑层面）

### 4.1 系统层次结构

```
用户界面层（UI）
 ├── 行程规划界面
 ├── 地图与发现界面
 └── 智能助手交互界面

业务逻辑层（Logic）
 ├── 行程数据管理
 ├── 兴趣点数据处理
 ├── 智能推荐与生成逻辑
 └── 用户交互与事件管理

数据层（Data）
 ├── 用户信息与行程数据
 ├── 兴趣点数据与缓存
 └── 外部API调用结果存储
```

---

### 4.2 数据流程概述

1. 用户输入目的地与日期；
2. 系统查询目的地POI与交通数据；
3. 调用规划算法或AI生成行程；
4. 用户可手动调整并保存；
5. 系统在地图展示并支持导航；
6. 智能助手可基于行程上下文进行对话与修改。

## 六、系统特点与优势

1. **智能性** ：引入大模型进行行程生成与语义交互；
2. **交互性** ：拖拽、地图、智能浮窗等多模态交互；
3. **扩展性** ：模块化设计，后续易接入更多AI与API；
4. **实用性** ：结合真实地图与出行服务，落地性强；
5. **用户体验** ：流程自然、操作直观、界面层次清晰。
